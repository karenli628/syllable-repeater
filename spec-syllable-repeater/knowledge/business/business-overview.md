# business-overview

## 業務目標

Syllable Repeater 幫單一學習者把英文音檔切成音節，從句尾開始逐步疊加跟讀，並在本機完成校正、播放、匯出、錄音比對、課件保存與 SRS 複習。v1.1 起支援完整製課流程：多句音檔先切段落（段落標籤）、再進單句分析、可增減音節切點、可把音節積木自由編排成自訂練習單元，最後整包封裝（`.abopack` v3）分享。產品價值不是生成新聲音，而是保留原聲並讓使用者用更細的音節單位、自訂的組合方式練習。

## 核心規則

| 規則 | 說明 |
|------|------|
| 原聲不可替換 | 播放/匯出音訊逐 sample 來自原始 PCM 切片，禁止 TTS/AI 合成/跨來源拼接；同 Lesson 同原音的多段切片可任意順序/次數串接（v1.1 補述）；Demucs 分析軌只供辨識 |
| 音節疊加 | 步數 = 音節總數（編輯後當時值，M11）；第 n 步 = 句尾倒數 n 個音節到句尾 |
| 金標準 | `She has excellent communication skills` = 11 音節、10 切點、11 步（未編輯預設） |
| 排列覆蓋 | 自訂排列 0 列＝完整單句 1 單元、N 列＝N 單元；判定入口唯一（M12）；自動模式演算法不可改 |
| 三層靜音 | 積木預設 1/1、整列預設 3/1；列靜音基準只算擺放積木原始長度一次；列尾不留、積木尾保留（M3 r6） |
| 語言路由 | ASR/切分器雙抽層（M13）；查無語言明確拒絕附支援清單，禁默默英文 fallback（M14） |
| 真實進度 | 進度與「已就緒」必須由真實工作量推進；假百分比＝功能錯誤（M15） |
| 本機隱私 | API key 只進 Keychain；錄音零持久化——無 RecordingBuffer、temp finally 清除、僅目前單元記憶體 PCM 五時機清除（M10 r7） |
| 授權白名單 | release bundle 不得含 GPL/AGPL/non-commercial/research-only；FFmpeg 必須 LGPL dynamic shared；新 ASR 引擎/模型上架必經五步審查 |

## 業務流程

1. 使用者匯入多句音檔，於段落標籤頁自動切段＋手動保留/捨棄，可存/載 `.abolabel`（同檔指紋提醒）。
2. 勾選區段送單句分析（或直接匯入單句）；就緒驗證後經語言路由產出音節時間軸。
3. 使用者校正切點（可增刪、改字）；總數變更即新步數基準，舊排列標過期。
4. 一鍵生成或自由編排積木/組塊/列；練習頁依單元播放，四態字稿/譯文顯示。
5. 錄音單次比對（只比單元原音一次）；只留記憶體回放，比對後磁碟錄音即清。
6. 使用者保存 `.abopack`（v3 可整包含標籤/排列/進度）與 `.aboprogress`，SRS 跨日零懲罰。
7. 匯出依四層選擇（音訊來源→排列來源→單元範圍→設定覆寫），來源指紋綁定不可混用。
8. 以未簽章 macOS x86_64 交付。

## 範圍外

手機端、Windows、雲端同步、伺服器、批次匯入、TTS/AI 合成音訊、金流、Apple 官方簽章/notarization、Apple Silicon/universal binary 均不在範圍。v1.1 另排除：跨 Lesson 積木拼接、跨單元錄音暫存（r7 撤回）、非英文切分器交付（僅留抽層架構）、線上 ASR（僅本地 sidecar）。
