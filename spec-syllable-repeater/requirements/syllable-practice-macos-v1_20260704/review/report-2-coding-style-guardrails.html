4. 紅線（違反即 blocking，不談判）

R1・M1 原聲鐵則：任何練習播放/匯出音訊只能來自 renderStep 的 copy＋串接路徑。禁止任何生成、合成、重採樣、音高改寫路徑進入播放或匯出；分析模組只讀不寫。CT-01 位元級回歸測試常駐 CI，動到 PracticeEngine 必先跑它。
R2・M5 Domain 純度：packages/domain 禁止 import flutter／infra／sidecar／dart:io（走 port 注入）。domain_purity_test 會擋，不要試圖繞過測試本身。
R3・M9/M10 合規與隱私：禁止 GPL/AGPL/非商用授權碼或模型進 release；FFmpeg 必須 LGPL 動態連結（release gate 會擋 --enable-gpl）。禁止任何 key/個資/錄音落地：key 只進 SecureStore、錄音刪除在 finally、schema 不開音訊欄位。
R4・三同步：規則改動＝需求成稿＋程式＋測試同步改，任何一者落後即視同核心被破壞（憲法 C10）。錯誤碼、schema、SRS 參數、靜音規則皆適用。涉及 2.5 核心的變更先過七題變更防線。
